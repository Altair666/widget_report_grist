# widget_report_grist

Custom widget для [Grist](https://www.getgrist.com/) — генерация отчётов из таблиц
по PDF-шаблонам с настраиваемым расположением полей.

## Возможности

**Основной экран**
- Панель «Последние использованные фильтры» с указанием автора фильтра (сохраняется при печати)
- Выбор даты (по умолчанию — *месяц прописью + год*, можно изменить через календарь)
- Выбор / создание / редактирование шаблона
- Экспорт таблицы в CSV
- Печать PDF по выбранному шаблону
- Фильтр по партии / заказу / свободный поиск

**Редактор шаблона**
- Загрузка PDF (через кнопку «Загрузить PDF»)
- Drag-and-drop трёх типов полей: *Конфигурация изделия*, *Серийный номер*, *Дата*
- Группы полей (1…N) — каждое поле помечено номером своей группы (`G1: Сер.№`, `G2: Дата`, …)
- Кнопки «Сбросить» и «Сохранить»

## Подключение в Grist

1. Захостите репозиторий (GitHub Pages → Settings → Pages → Branch: main → /).
   Виджет окажется по адресу `https://altair666.github.io/widget_report_grist/`.
2. В документе Grist: добавьте новый виджет → **Custom**.
3. Вставьте URL виджета, выберите источник данных (таблицу).
4. Дайте права **read table**.

## Структура

- `index.html` — точка входа, всё в одном файле (HTML + CSS + JS)
- Зависимости подключаются с CDN:
  - `grist-plugin-api.js` — связь с Grist
  - `pdf.js` — рендер PDF-шаблонов
- Шаблоны сохраняются в `localStorage` браузера
  (мигрировать в Grist-таблицу — TODO, см. функции `loadTemplates()` / `saveTemplates()`)

## Локальная разработка

```bash
python3 -m http.server 8000
```

В Grist указать `http://localhost:8000`.
