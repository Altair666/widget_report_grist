# widget_report_grist — handoff context for Claude Code

Полный контекст разработки виджета, чтобы продолжить работу в Claude Code CLI.

---

## 0. Краткое резюме

- **Что:** custom widget для [Grist](https://www.getgrist.com/) — генерация отчётов из таблиц по PDF-шаблонам с настраиваемым расположением полей через drag-and-drop.
- **Репозиторий:** https://github.com/Altair666/widget_report_grist
- **Live (GitHub Pages):** https://altair666.github.io/widget_report_grist/
- **Целевой Grist-документ:** https://lmp-test-dec.getgrist.com/5GtDphApyrjG/LMP-ESKD
- **Текущая версия:** `v0.3.0`
- **Стек:** один `index.html` (HTML + CSS + vanilla JS), `pdf.js` с CDN, `grist-plugin-api.js` с CDN.

## 2. Полная спецификация виджета (как было запрошено)

### 2.1 Основная страница

Сверху вниз:

1. **Окно «Последние использованные фильтры»** — сохраняется после клика «печать», показывает кто фильтр сделал.
2. **Строка 1 кнопок:** слева — поле даты (с календарём), справа от него — «выбрать шаблон», далее — «создать шаблон», справа края — «редактировать шаблон».
3. **Строка 2 кнопок:** под «создать шаблон» того же размера — **«экспорт таблицы»**, под «редактировать шаблон» того же размера — **«печать PDF»**.
4. **Заголовок с фильтрами:** выбор партии / конкретного заказа.
5. **Под ним** — то, что выпадает из фильтра (таблица результатов).

### 2.2 Страница создания/редактирования шаблона

- Слева сверху поле «**конфигурация изделия**», справа от него — числовое поле (кол-во групп).
- На этом же уровне справа листа — кнопки «**Сбросить**» и «**Сохранить**».
- Следующая строка под конфигурацией — «**серийный номер**».
- Под серийным номером — «**дата**».
- Справа от поля «дата» — кнопка «**загрузить PDF**».
- Под полями даты и кнопкой загрузить PDF на оставшуюся страницу — область отображения PDF.

### 2.3 Логика редактора шаблона

- Загружаешь PDF по кнопке — он появляется в области отображения в нормальном виде.
- Размещаешь поверх PDF через drag-and-drop поля: конфигурация / серийный / дата.
- Числовое поле задаёт **кол-во групп**. Если поставить 2 — будут группы 1 и 2, в каждой по своему набору (1 конфиг + 1 серийный + 1 дата, 2 конфиг + 2 серийный + 2 дата).
- При размещении должно быть **чётко понятно**, к какой группе относится поле.
- Кнопка «Сохранить» — запоминает расположение полей.
- Кнопка «Сбросить» — сбрасывает все установленные поля.

### 2.4 Логика основной страницы

- **Дата** — по умолчанию сегодняшняя в формате *месяц прописью на русском + год 4-мя символами* (например, «июнь 2026»). Можно поставить любую через календарь.
- **Выбрать шаблон** — модалка со списком сохранённых шаблонов, выбираешь → OK / Отмена.
- **Создать шаблон** / **Редактировать шаблон** — открывают редактор. «Редактировать» неактивна (серая), если шаблон не выбран.
- **Экспорт таблицы** — экспорт списка в CSV.
- **Печать PDF** — также неактивна, если шаблон не выбран.

### 2.5 Дополнительно (v0.2 / v0.3)

- Подписывать **версию документа** на странице.
- Кнопка печати должна быть серой если не выбран шаблон.
- Активная группа перенесена внутрь блока «Конфигурация изделия» справа.
- «Кол-во групп» вынести строкой над блоком.
- Блоки «Конфигурация изделия» и «Имя шаблона» — одинаковой высоты.
- «Назад» в редакторе переименовать в «**Отменить**».
- Кнопки «Сохранить», «Сбросить», «Отменить» — одинакового размера, расположены вертикально в порядке: Сохранить → Сбросить → Отменить.
- В бэйдже версии — только версия, без даты.


### Структуры данных (как хранятся сейчас в localStorage)

```javascript
// Шаблон
{
  id:         'id-abc123',
  name:       'Бланк ОТК',
  groupCount: 2,
  fields: [
    { id:'id-...', type:'config'|'serial'|'date', group:1, page:1, x:0.42, y:0.31 }
  ],
  pdfBase64:  'JVBERi0xLj...',     // весь PDF
  updatedAt:  1718380000000
}

// Запись последнего фильтра
{
  label:  'партия: A-12, заказ: ORD-007',
  author: 'неизвестный пользователь',
  ts:     1718380000000
}
```

`x`, `y` хранятся как **доли** размера страницы (0..1) — устойчиво к смене zoom/scale при рендере.

### Точки расширения (где менять при добавлении фичи)

| Функция | Что делает |
|---|---|
| `loadTemplates()` / `saveTemplates()` | заменить на чтение/запись в Grist таблицу `Templates` |
| `pushLastFilter()` / `loadLastFilters()` | заменить на работу с таблицей `LastFilters` |
| `printPdf()` | сейчас просто открывает PDF в новой вкладке — здесь нужно `pdf-lib` для впечатывания значений |
| `populateFilterDropdowns()` | автодетект колонок — расширить под реальные имена таблицы Grist |
| `initGrist()` | здесь добавить чтение текущего пользователя и подписки на источник данных |

## 5. Служебные таблицы Grist (предполагаемая схема)

Скрипт `scripts/create_grist_tables.sh` создаёт две таблицы в документе `5GtDphApyrjG` на `lmp-test-dec.getgrist.com`. **На момент handoff ещё не выполнен** — мой sandbox блокировал хост (`x-deny-reason: host_not_allowed`). В Claude Code (с обычным интернетом) запустится без проблем.

### Templates

| Колонка | Тип | Назначение |
|---|---|---|
| `ClientId`   | Text          | id из виджета (для маппинга localStorage ↔ Grist) |
| `Name`       | Text          | название шаблона |
| `GroupCount` | Int           | кол-во групп |
| `Fields`     | Text          | JSON массив размещённых полей |
| `PdfBase64`  | Text          | PDF в base64 |
| `UpdatedAt`  | DateTime:UTC  | время изменения |
| `UpdatedBy`  | Text          | автор |

### LastFilters

| Колонка | Тип | Назначение |
|---|---|---|
| `Label`        | Text          | описание фильтра |
| `Author`       | Text          | кто применил |
| `BatchValue`   | Text          | значение партии |
| `OrderValue`   | Text          | значение заказа |
| `SearchValue`  | Text          | строка поиска |
| `TemplateName` | Text          | имя шаблона (если запись при печати) |
| `AppliedAt`    | DateTime:UTC  | время применения |

## 6. Открытые задачи (по приоритету)

1. **Создать таблицы** — запустить `bash scripts/create_grist_tables.sh` (нужны Grist API key + сетевой доступ).
2. **Миграция localStorage → Grist** — переписать `loadTemplates / saveTemplates / pushLastFilter / loadLastFilters` на `grist.docApi.fetchTable('Templates')` и `grist.docApi.applyUserActions([['AddRecord','Templates',null,{...}]])`. Учесть, что `PdfBase64` может быть очень большим — возможно, разбивать на чанки или хранить в attachment-колонке.
3. **«Впечатывание» значений в PDF при печати** — добавить `pdf-lib`, в `printPdf()`:
   - открыть PDF из шаблона,
   - для каждого `placedField`: взять координаты, посчитать в координатах страницы PDF (учесть DPI/units), нарисовать текст значением из выбранной строки Grist;
   - сохранить, открыть в новой вкладке.
   Где брать значения для каждой группы — обсудить (выделенная строка? первые N строк отфильтрованной таблицы?).
4. **Имя текущего пользователя** — через `grist.docApi` или `requiredAccess:'full'` + `grist.api`. Сейчас в `state.filterAuthor = 'неизвестный пользователь'`.
5. **Привязка к источнику данных Grist** — автодетект колонок «партия/заказ» работает, но имена могут отличаться в реальной таблице LMP-ESKD. Возможно, добавить настройки виджета через `grist.setOptions`.
6. **Чанкование PDF в Grist** — поле Text в Grist имеет лимит, большие PDF (>1 МБ) могут не сохраниться. Альтернатива — attachment-колонка через `/api/docs/{docId}/attachments`.
7. **Множественный выбор и групповая печать** — печать всех групп за один PDF (1 страница на группу или несколько групп на одну страницу шаблона).

## 7. Команды для разработки

```bash
# Клонировать
git clone https://github.com/Altair666/widget_report_grist.git
cd widget_report_grist

# Локальный запуск (любой статический сервер)
python3 -m http.server 8000
# затем в Grist: Add Widget → Custom → URL = http://localhost:8000

# Создать служебные таблицы (сначала export GRIST_API_KEY=...)
bash scripts/create_grist_tables.sh

# Релизный коммит — бампить VERSION в index.html и тегать
git commit -am "vX.Y.Z: ..."
git tag vX.Y.Z
git push --tags
```

GitHub Pages обновляется автоматически при push в `main` (~30-60 секунд).
